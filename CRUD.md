# 주제 : 머신러닝 모델 실험 이력 로그
- 머신러닝 학습, 예측 과정에서 발생하는 결과를 기록하고 관리
- 도메인 설계 - 사용한 알고리즘, 하이퍼파라미터값, 실험결과(정확도, 손실률) 등을 저장
C : 새로운 결과에서의 실험데이터 삽입
R : 과거 실험 결과, 설정값 조회
U : 과거 실험의 설정값 수정, 추가적인 평가지표 업데이트
    - 실험에 대한 메모나 태그값 추가/수정
D : 유효하지 않은 실험의 기록, 데이터 삭제(soft delete)

역할분담
태현 : C
윤서 : R
기연 : U
찬영 : D

컬럼, 값 예시
날짜/시간	알고리즘	하이퍼파라미터값	정확도	손실률	…..	메모/태그
26.05.18 10:42	KNN		50%	40%		 


머신러닝 학습 코드는 간단하게 설정이 가능하고 빠른 결과가 나오는 난이도가 높지 않은 걸로 선택함 tensorflow
- 머신러닝이 메인이 아니라 spring이 메인이라 그럼

우리가 머신러닝 학습을 일단 시켜야 되는데 python 으로 시킬거아냐? 
python 이랑 spring이랑 어케 연결함

REST API로 분리
Python 쪽을 별도의 API 서버로 띄우고, Spring이 HTTP로 호출하는 방식이야.
[Spring Boot]  ──HTTP──>  [Python ML 서버 (FastAPI/Flask)]
   CRUD, DB              학습/예측 담당

Python: FastAPI(추천) 또는 Flask로 /train, /predict 엔드포인트 만들기
Spring: RestTemplate 또는 WebClient로 그 엔드포인트 호출
ML 모델은 Python 서버가 메모리에 들고 있거나 joblib/pickle로 로드


tensorflow로 빠르게 결과만 가져오는걸로.
알고리즘, 옵티마이저를 하나로 정해놓고 하는걸로.
이게 여러개가 되면 하이퍼파라미터 값이 너무 많아져서 테이블에 null이 너무 많을수있다
