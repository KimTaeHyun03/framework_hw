## 아키텍처

**머신러닝 실험 이력**(알고리즘, 하이퍼파라미터, 정확도, 손실률, 메모/태그)을 기록하는 2-서버 시스템입니다.

```
[사용자] → [Spring Boot]  ──HTTP/JSON──▶  [Python ML 서버]
              CRUD + DB    ◀──JSON──────    학습 / 예측
                │
                ▼
           [H2 Database]
```

- **Spring Boot (`spring-server/`)** 가 *메인* 결과물입니다. 실험 로그 도메인, CRUD, H2 영속화(Spring Data JPA)를 담당하고, `WebClient`로 Python 서버 호출을 오케스트레이션합니다.
- **Python ML 서버 (`ml-server/`)** 는 `/train`, `/predict`를 노출하는 *별도의* HTTP 서버(FastAPI)입니다. 학습/예측만 담당합니다. ML은 의도적으로 핵심이 아니며 — Spring이 핵심입니다.
- 두 서버는 **오직 REST(HTTP + JSON)** 로만 통신하며, DB나 프로세스를 공유하지 않습니다.

## 핵심 설계 제약 (팀 합의 없이 위반 금지)

- **알고리즘 + 옵티마이저를 하나로 고정합니다.** 여러 알고리즘을 지원하면 하이퍼파라미터 컬럼이 폭증하고 실험 로그 테이블이 NULL로 가득 차게 됩니다. 모델 설정은 최소·고속으로 유지합니다.
- **삭제는 물리 삭제가 아니라 soft delete**입니다.
- ML 프레임워크 표기가 설계 문서마다 다릅니다: `CRUD.md`는 **TensorFlow**, SVG는 **PyTorch**라고 적혀 있습니다. 어느 쪽으로든 코드를 커밋하기 전에 팀과 확인하세요.

## CRUD 역할 분담

각 CRUD 연산은 팀원별로 담당이 다릅니다. 특정 팀원을 대신해 작업할 때는 해당 연산 범위로 변경을 한정하세요:

| 연산 | 범위 | 담당 |
| --- | --- | --- |
| Create | 새 실험 결과 삽입 | 태현 |
| Read | 과거 결과 / 설정값 조회 | 윤서 |
| Update | 설정 수정, 평가지표 추가, 메모/태그 수정 | 기연 |
| Delete | 유효하지 않은 기록 soft delete | 찬영 |
