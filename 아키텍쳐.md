# 아키텍처 문서

> 26년도 3학년 1학기 프레임워크 팀과제
> 주제: **머신러닝 모델 실험 이력 로그** 관리 시스템

## 1. 개요

머신러닝 학습 / 예측 과정에서 발생하는 결과(알고리즘, 하이퍼파라미터, 정확도, 손실률 등)를
기록하고 관리하는 시스템이다.

- **Spring Boot**가 메인이며, CRUD와 DB 영속화를 담당한다.
- **Python ML 서버**는 실제 학습 / 예측만 담당하는 별도 서버로 분리한다.
- 두 서버는 **REST API(HTTP + JSON)** 로 통신한다.

머신러닝 자체가 과제의 핵심이 아니라 Spring이 핵심이므로, ML 모델은 설정이 간단하고
빠르게 결과가 나오는 난이도가 낮은 구성으로 선택한다.

# 2. 전체 구조

```
[사용자]
   │  학습/조회 요청
   ▼
[Spring Boot]            ──HTTP 요청──▶   [Python ML 서버]
  · CRUD + WebClient                       · FastAPI
  · 실험 로그 도메인       ◀──JSON 결과──    · 학습 / 예측
   │
   │ 실험 로그 저장
   ▼
[H2 Database]
```

흐름:

1. 사용자가 Spring Boot로 학습/조회 요청을 보낸다.
2. Spring Boot가 `WebClient`로 Python ML 서버의 엔드포인트를 HTTP 호출한다.
3. Python 서버가 학습/예측을 수행하고 결과를 JSON으로 응답한다.
4. Spring Boot가 그 결과를 실험 로그로 H2 Database에 저장한다.

> 다이어그램 원본: [`python_ml_spring_boot_architecture.svg`](./python_ml_spring_boot_architecture.svg)

## 3. 구성 요소

### 3.1 Spring Boot 서버 (`spring-server/`)

- **역할**: 실험 로그 도메인의 CRUD, DB 영속화, Python 서버 호출 오케스트레이션
- **주요 기술**
  - Spring Boot
  - Spring Data JPA
  - `WebClient` (Python ML 서버 HTTP 호출)
  - H2 Database

### 3.2 Python ML 서버 (`ml-server/`)

- **역할**: 모델 학습(`/train`), 예측(`/predict`) 담당
- **주요 기술**
  - FastAPI (또는 Flask) — `/train`, `/predict` 엔드포인트 제공
  - 머신러닝 프레임워크 (TensorFlow)
  - 모델은 메모리에 상주시키거나 `joblib`/`pickle` 등으로 로드
- **설계 원칙**: 알고리즘과 옵티마이저를 하나로 고정한다.
  여러 알고리즘을 허용하면 하이퍼파라미터 컬럼에 NULL이 과도하게 늘어나기 때문이다.

## 4. 도메인 — 실험 로그

| 항목 | 설명 | 예시 |
| --- | --- | --- |
| 날짜/시간 | 실험 수행 시각 | 26.05.18 10:42 |
| 알고리즘 | 사용한 알고리즘 | KNN |
| 하이퍼파라미터 | 모델 설정값 | (모델별) |
| 정확도 | 평가지표 | 50% |
| 손실률 | 평가지표 | 40% |
| 메모/태그 | 실험 설명 | (자유) |
| 삭제 여부 | soft delete 플래그 | - |

> 컬럼 상세 예시는 [`테이블 예시.xlsx`](./테이블%20예시.xlsx) 참고.

## 5. CRUD 기능 및 역할 분담

| 기능 | 설명 | 담당 |
| --- | --- | --- |
| **C**reate | 새로운 실험 결과 데이터 삽입 | 태현 |
| **R**ead | 과거 실험 결과 / 설정값 조회 | 윤서 |

| **U**pdate | 설정값 수정, 평가지표 / 메모·태그 추가·수정 | 기연 |
| **D**elete | 유효하지 않은 실험 기록 삭제 (**soft delete**) | 찬영 |
